# 飞书上下文同步规范

## Purpose

本规范用于定义飞书工作上下文的只读采集、SQLite 规范写入、完整增量区间、稳定去重、有边界分页和按来源故障隔离，并将可靠采集与异步分析解耦。

## Requirements

### Requirement: 只读用户身份
飞书适配器 MUST 使用用户身份调用受支持的读取命令，并 MUST NOT 调用飞书修改命令。

#### Scenario: 构建一次同步运行
- **WHEN** 用户启动飞书同步
- **THEN** 每条生成的命令都使用 `--as user`，并属于已配置的只读命令白名单

### Requirement: 相关上下文采集
适配器 SHALL 采集提及用户的非机器人群消息、非机器人 P2P 对话双方的消息、配置滚动窗口内的日历日程，以及分配给用户且尚未完成的任务，并将标准化来源与上游任务写入 SQLite。系统 MUST 在持久化前排除机器人发送的消息、与机器人的整个 P2P 会话和已完成任务。

#### Scenario: 同步所有已启用的飞书来源
- **WHEN** 所有飞书命令都返回有效的非机器人记录和未完成任务
- **THEN** 系统在 SQLite 中写入标准化提及、P2P、日历来源和上游任务，并使用提供方范围内稳定的来源 ID

#### Scenario: 上游意外返回已完成任务
- **WHEN** 任务查询已要求只返回未完成任务，但上游响应仍包含已完成记录
- **THEN** 标准化阶段丢弃该记录，并且不为其创建或更新来源、上游任务或 Todo Markdown

#### Scenario: 排除机器人 P2P 会话
- **WHEN** P2P 搜索返回机器人对端发送的消息或用户发送给机器人对端的消息
- **THEN** 系统不持久化这些消息、不为其发现人物，也不把它们计入同步记录数

### Requirement: 增量且幂等的同步
适配器 SHALL 使用 SQLite 游标、完整增量区间、近期滚动校准、应用层受控分页和稳定来源 ID。每次运行 MUST 覆盖上次成功游标至本次同步时间的全部区间，并将查询起点向前扩展至配置的滚动校准下界；校准窗口 MUST NOT 截短较长的增量区间。消息分页的每个成功页面 MUST 在请求下一页前完成规范化、持久化和幂等入队；重复运行不得创建重复来源或重复分析任务。

#### Scenario: 重复执行存在重叠的运行
- **WHEN** 第二次运行返回第一次已经采集的记录，并额外返回一条新记录
- **THEN** 系统更新或跳过现有记录，只创建一条新来源，并只为需要分析的内容幂等入队

#### Scenario: P2P 分页中途失败后重试
- **WHEN** P2P 窗口的第一页已经持久化，但后续页面读取失败，并且用户再次启动同步
- **THEN** 系统保留第一页来源、重新安全读取未完成窗口、不创建重复来源，并在所有页面完成后推进 P2P 来源游标

#### Scenario: 同步间隔超过滚动校准时长
- **WHEN** 上次成功游标距本次同步超过配置的滚动校准时长
- **THEN** 系统仍从上次成功游标开始读取，完整覆盖两次同步之间的全部时间，而不是只读取近期校准窗口

#### Scenario: 消息在游标推进后才可见
- **WHEN** 一条创建时间早于当前游标的消息未出现在首次成功搜索中，但在滚动校准期内的后续搜索才可见
- **THEN** 系统在后续同步中按稳定消息 ID 补采该来源并幂等创建分析任务

### Requirement: 有边界的回填
适配器 SHALL 将初始消息回填拆分为可配置的时间窗口，并由应用逐页消费每个窗口，直到上游明确返回 `has_more=false`。系统 MUST 在缺少下一页游标、游标重复、达到安全页数上限或单页失败时把窗口标记为未完成，并且 MUST NOT 推进对应来源游标。

#### Scenario: 回填范围超过一个结果窗口
- **WHEN** 初始日期范围跨越多个已配置窗口
- **THEN** 适配器分别请求每个窗口，并且只在来源的全部窗口与页面完成后推进来源游标

#### Scenario: 单个窗口超过 CLI 全量分页上限
- **WHEN** P2P 窗口包含超过四十页消息，并且每一页都返回有效的下一页游标
- **THEN** 系统继续逐页请求直到 `has_more=false`，每页成功后立即持久化，不受 `--page-all` 聚合上限影响

#### Scenario: 上游声明有下一页但没有有效游标
- **WHEN** 消息页面返回 `has_more=true`，但 `page_token` 缺失、为空或重复
- **THEN** 系统明确标记窗口与来源失败，保留已经持久化的页面，并且不把截断结果报告为成功

#### Scenario: 分页达到安全页数上限
- **WHEN** 消息窗口达到配置的安全页数上限后仍返回 `has_more=true`
- **THEN** 系统停止继续请求、报告分页未完成、保留已经持久化的页面，并且不推进来源游标

### Requirement: 失败隔离与状态
适配器 SHALL 在 SQLite 中按同步运行及来源报告成功、失败、数量、时间戳和可操作错误，同时保留之前有效的游标；运行期间 SHALL 公开当前阶段、来源、窗口、分页、采集和持久化数量。权限、认证和参数错误 MUST 保留缺失 scope、错误代码、日志 ID、官方处理入口以及可用的 CLI 升级提示；同步结果 MUST 与异步分析结果分开表示。

#### Scenario: 日历获取失败
- **WHEN** 消息和任务同步成功，但日历命令失败
- **THEN** 系统提交成功来源及其分析任务、展示日历失败，并且不推进日历游标

#### Scenario: 飞书用户身份缺少权限
- **WHEN** 任一只读命令返回权限不足、认证失效或缺失 scope 的结构化错误
- **THEN** 对应来源被标记为失败，Settings 显示需要人工处理的提醒、缺失 scope 和官方处理入口，并且系统不自动授权或升级

#### Scenario: 飞书返回字段校验错误
- **WHEN** 任一只读命令返回 `99992402` 或其他参数错误，并提供日志 ID 或排查链接
- **THEN** 对应来源被标记为参数错误，Settings 展示错误代码和可用的排查入口，而不是只展示完整子进程命令

#### Scenario: 查看运行中的同步
- **WHEN** 同步正在采集某个来源的时间窗口和分页
- **THEN** 状态 API 返回该来源、窗口序号、页序号、当前数量和最近更新时间

#### Scenario: 分析 Provider 不可用
- **WHEN** 来源已可靠提交但异步分析随后失败
- **THEN** 同步运行仍保持采集成功，分析任务独立展示失败并允许重试

### Requirement: 飞书兼容的查询参数
适配器 MUST 为飞书消息和日历查询生成不含毫秒的秒级 ISO 时间，并 SHALL 在任务查询中显式请求未完成任务。

#### Scenario: 构造消息回填窗口
- **WHEN** 同步服务为一个时间窗口生成消息搜索命令
- **THEN** `--start` 和 `--end` 使用 `YYYY-MM-DDTHH:mm:ssZ` 格式，不包含毫秒

#### Scenario: 构造任务查询
- **WHEN** 同步服务请求分配给当前用户的任务
- **THEN** 命令包含 `--complete=false` 和分页参数

### Requirement: 来源提交后异步分析
同步器 SHALL 只负责采集、规范化并把来源及其持久分析任务可靠提交到 SQLite，不得在同步调用链内等待 LLM。分析 Worker SHALL 从本地持久队列异步领取任务，并按配置容量构造批次；单个来源采集失败 MUST NOT 阻止其他已成功来源提交分析任务。

#### Scenario: 多个来源成功拉取
- **WHEN** 一轮同步取得人物、提及、P2P、日历和任务记录
- **THEN** 系统提交全部规范来源、上游任务和幂等分析任务后结束同步，Worker 随后对提及和 P2P 消息执行有边界的批量分析

#### Scenario: 部分来源失败
- **WHEN** 日历拉取失败，但提及和 P2P 消息已成功拉取并落盘
- **THEN** 系统保留日历失败状态，同时提交已取得消息的分析任务，并由 Worker 独立记录异步分析结果
