# 持久分析队列规范

## Purpose

本规范用于定义 SQLite 持久分析队列的幂等入队、至少一次交付、租约恢复、有界重试和原子结果提交。

## Requirements

### Requirement: 来源提交时持久入队
系统 SHALL 在提交可分析来源的同一 SQLite 事务中创建持久分析任务，并使用确定性幂等键防止重复任务。

#### Scenario: 重叠同步返回已有消息
- **WHEN** 重叠同步再次提交内容和有效分析配置均未变化的消息
- **THEN** 系统复用现有成功任务或候选结果，不创建重复分析任务

### Requirement: 至少一次交付与租约恢复
分析 Worker SHALL 通过有期限租约领取任务，V1 MUST 同时只处理一个任务；租约到期且未完成的任务 MUST 能够被重新领取。

#### Scenario: Worker 在分析中崩溃
- **WHEN** Worker 领取任务后在提交结果前退出且租约到期
- **THEN** 任务恢复为可领取状态，并可由重启后的 Worker 再次处理

### Requirement: 有界重试与终态失败
系统 SHALL 将失败区分为可重试与终态失败，对可重试失败执行有次数上限的退避，并在每次状态变化时保存脱敏错误。

#### Scenario: Provider 连续超时
- **WHEN** 一个任务达到配置的最大重试次数仍然超时
- **THEN** 系统将任务标记为 `failed_terminal`，停止自动调用并允许用户显式重试

### Requirement: 分析成功原子提交
系统 MUST 在单个 SQLite 事务中提交分析运行、全部候选与证据并完成任务；任一候选校验或写入失败时 MUST NOT 暴露部分结果。

#### Scenario: 批量结果包含非法候选
- **WHEN** Provider 输出中的一个候选未通过领域校验
- **THEN** 系统不保存该批次的任何候选，并把任务记录为失败
