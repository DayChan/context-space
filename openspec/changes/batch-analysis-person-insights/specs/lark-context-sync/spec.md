## ADDED Requirements

### Requirement: 拉取完成后批量分析
同步器 SHALL 将来源采集持久化与 LLM 派生分析划分为两个阶段，先完成本轮所有可用来源的拉取、规范化和落盘，再把提及与 P2P 消息按配置容量批量交给 LLM。单个来源采集失败 MUST NOT 阻止其他已成功来源进入后续分析。

#### Scenario: 多个来源成功拉取
- **WHEN** 一轮同步取得人物、提及、P2P、日历和任务记录
- **THEN** 系统先持久化全部规范来源，再统一映射原生任务并对提及和 P2P 消息执行有边界的批量分析

#### Scenario: 部分来源失败
- **WHEN** 日历拉取失败，但提及和 P2P 消息已成功拉取并落盘
- **THEN** 系统保留日历失败状态，同时仍对已取得的消息执行批量分析并独立记录分析结果
