## MODIFIED Requirements

### Requirement: 展示来源依据的详情视图
详情视图 SHALL 展示管理模式、来源引用、适用时的置信度，并区分生成内容与用户拥有的内容。People 详情中的 Provenance SHALL 将可解析的消息引用展示为具体的消息标题、时间、正文和详情链接，而不是仅展示不透明 ID。

#### Scenario: 查看混合管理文档
- **WHEN** 用户打开混合管理的人物档案或 Todo
- **THEN** UI 在视觉上区分可编辑的用户内容与有证据支撑的生成内容

#### Scenario: 查看人物来源消息
- **WHEN** 用户打开含有消息来源引用的人物详情
- **THEN** UI 直接展示每条来源消息的时间与内容，并允许打开对应来源详情

### Requirement: 同步状态可见
Settings 和 Now SHALL 展示飞书可用性、上次运行状态、各来源结果以及手动只读同步触发入口。Settings SHALL 提供运行中的同步状态窗口，展示当前阶段、来源、窗口、分页、数量和错误，并在运行结束后保留最终结果。

#### Scenario: 展示局部同步失败
- **WHEN** 一次同步完成但其中一个来源失败
- **THEN** 页面同时展示成功数量和失败来源的消息

#### Scenario: 观察运行中的同步
- **WHEN** 用户触发手动同步
- **THEN** 状态窗口持续更新当前采集或分析步骤，并在发生错误时展示可操作的错误消息

## ADDED Requirements

### Requirement: Todo 状态交互
工作台 SHALL 在 Todo 列表、Now 概览和详情中提供可访问的状态控件，用于标记完成或重新打开，并在失败时恢复原状态和展示错误。

#### Scenario: 从列表完成 Todo
- **WHEN** 用户点击开放 Todo 的完成控件且 API 保存成功
- **THEN** 控件变为已完成状态，详情与后续加载读取同一持久化结果
