## MODIFIED Requirements

### Requirement: 相关上下文采集
适配器 SHALL 采集提及用户的群消息、P2P 对话双方的消息、配置的滚动窗口内的日历日程，以及分配给用户且尚未完成的任务。系统 MUST NOT 将已完成的飞书任务写入新的来源或 Todo 文档。

#### Scenario: 同步所有已启用的飞书来源
- **WHEN** 所有飞书命令都返回有效记录
- **THEN** 系统写入标准化的提及、P2P、日历和未完成任务来源 Markdown 文档，并使用提供方范围内稳定的来源 ID

#### Scenario: 上游意外返回已完成任务
- **WHEN** 任务查询已要求只返回未完成任务，但上游响应仍包含已完成记录
- **THEN** 标准化阶段丢弃该记录，并且不为其创建或更新来源与 Todo 文档

### Requirement: 失败隔离与状态
适配器 SHALL 按来源报告成功、失败、数量、时间戳和可操作的错误消息，同时保留之前有效的检查点。对于权限不足或认证错误，状态 MUST 保留缺失 scope、官方处理提示和可用的控制台链接；对于参数错误，状态 MUST 保留错误代码、日志 ID 和排查链接；如果错误信封包含 CLI 升级通知，状态 SHALL 同时展示升级命令和版本信息。

#### Scenario: 日历获取失败
- **WHEN** 消息和任务同步成功，但日历命令失败
- **THEN** 系统持久化成功的来源，在同步状态中展示日历失败，并且不推进日历检查点

#### Scenario: 飞书用户身份缺少权限
- **WHEN** 任一只读命令返回权限不足、认证失效或缺失 scope 的结构化错误
- **THEN** 对应来源被标记为失败，Settings 显示需要人工处理的提醒、缺失 scope 和官方处理入口，并且系统不自动授权或升级

#### Scenario: 飞书返回字段校验错误
- **WHEN** 任一只读命令返回 `99992402` 或其他参数错误，并提供日志 ID 或排查链接
- **THEN** 对应来源被标记为参数错误，Settings 展示错误代码和可用的排查入口，而不是只展示完整子进程命令

## ADDED Requirements

### Requirement: 飞书兼容的查询参数
适配器 MUST 为飞书消息和日历查询生成不含毫秒的秒级 ISO 时间，并 SHALL 在任务查询中显式请求未完成任务。

#### Scenario: 构造消息回填窗口
- **WHEN** 同步服务为一个时间窗口生成消息搜索命令
- **THEN** `--start` 和 `--end` 使用 `YYYY-MM-DDTHH:mm:ssZ` 格式，不包含毫秒

#### Scenario: 构造任务查询
- **WHEN** 同步服务请求分配给当前用户的任务
- **THEN** 命令包含 `--complete=false` 和分页参数
