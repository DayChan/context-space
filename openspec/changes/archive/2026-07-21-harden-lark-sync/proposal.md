## Why

当前飞书同步会拉取已完成任务，消息查询又会把带毫秒的 ISO 时间传给飞书，导致 `99992402 field validation failed`。同时，`lark-cli` 返回权限不足、认证失效或升级提示时，系统只保存一段不易处理的命令错误，用户无法快速判断下一步操作。

## What Changes

- 飞书任务查询显式限定为未完成任务，并在标准化阶段再次过滤已完成记录。
- 飞书消息与日历查询统一使用不含毫秒的秒级 ISO 时间，避免字段校验失败。
- 解析 `lark-cli` 的结构化错误信封，保留错误类型、代码、缺失权限、处理提示、控制台链接、排查链接和升级通知。
- 在同步状态与 Settings 中针对权限或认证问题展示显著、可操作的处理提醒，并区分参数错误与普通命令失败。
- 同步部分失败时不再提示“同步完成”，而是明确告知存在需要处理的来源。
- 为任务过滤、时间格式、错误解析、状态持久化和前端提醒增加自动化测试。

## 能力

### 新增能力

无。

### 修改能力

- `lark-context-sync`：补充未完成任务过滤、飞书兼容时间格式，以及权限、认证、参数和 CLI 升级错误的可操作提醒要求。

## 影响

- 修改飞书命令构造、错误解析、标准化与同步状态模型。
- 扩展本地 API 返回的同步来源结果，但保持现有字段兼容。
- 修改 Settings 同步反馈和错误详情展示。
- 不新增外部依赖，不改变只读命令白名单，也不会自动执行授权或升级命令。
