## MODIFIED Requirements

### Requirement: 相关上下文采集
适配器 SHALL 采集提及用户的群消息、非机器人 P2P 对话双方的消息、配置的滚动窗口内的日历日程，以及分配给用户的任务。适配器 MUST 在持久化前排除机器人发送的消息和与机器人的整个 P2P 会话。

#### Scenario: 同步所有已启用的飞书来源
- **WHEN** 所有飞书命令都返回有效的非机器人记录
- **THEN** 系统写入标准化的提及、P2P、日历和任务来源 Markdown 文档，并使用提供方范围内稳定的来源 ID

#### Scenario: 排除机器人 P2P 会话
- **WHEN** P2P 搜索返回机器人对端发送的消息或用户发送给机器人对端的消息
- **THEN** 系统不持久化这些消息、不为其发现人物，也不把它们计入同步记录数

### Requirement: 失败隔离与状态
适配器 SHALL 按来源报告成功、失败、数量、时间戳和可操作的错误消息，同时保留之前有效的检查点。同步服务 SHALL 在运行期间公开当前阶段、来源、窗口、分页、采集和持久化数量，并在失败时公开错误。

#### Scenario: 日历获取失败
- **WHEN** 消息和任务同步成功，但日历命令失败
- **THEN** 系统持久化成功的来源，在同步状态中展示日历失败，并且不推进日历检查点

#### Scenario: 查看运行中的同步
- **WHEN** 同步正在采集某个来源的时间窗口和分页
- **THEN** 状态 API 返回该来源、窗口序号、页序号、当前数量和最近更新时间
